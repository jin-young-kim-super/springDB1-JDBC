세션 1과 세견 2가 실제 데이베 베이스의 money라는 칼럼의 데이터를 수정하는 쿼리가 각각의 트랜잭션에 있다고 하자.
근데 만약에 세션 1이 트랜잭션이 종료가 안 돼서 실제 DB의 money 데이터를 수정 완료하지도 않았는데 세션 2가 money 데이터를 수정을 해버리게 되면 
여러가지 격리성을 위배하여 여러가지 심각한 문제가 초래한다. 
이러한 문제를 해결하기 위한 것이 DB 락이다. 
결론부터 말하터 수정쿼리문은 DB락을 획득해야만 실행시킬 수가 있다(물론, commit이 되기 전까지는 해당 세션 테이블에서만 수정이 일어남)
수정이 완료되고 나면 세션 1은 DB락을 반환하여, 세션 2가 그 락을 획득하여 쿼리문을 실행시킬 수가 있다. 
세션 2는 DB에 락이 반환될 떄까지 대기해야 한다(이떄 무한정 대기하는 것이 아니라, 롹 타임아웃 설정 시간동안만 대기를 한다)
DB의 락은 트랜잭션이 commit/rollback에 의해서 종료가 되어야만 반환이 된다. 
(참고로, 여기서의 변경은 등록/수정/삭제를 말한다, 왜냐하면 등록/삭제 또한 데이터를 변경하는 작업(DML)이기 떄문이다.읽기 시에는 MVCC에 의해 직전의 실제 데이터베이스의 데이터를 DB락이 없이 일어 들일 수 있기에 락을 필요로 하지 않는다.)


더 정확한 원리
Buffer Pool vs Undo Log
결론 : Buffer Pool은 메모리 위의 최신 commit된 데이터를 캐싱하고 있으며, 항상 최신의 데이터를 보존하고 있다.(DB가 자주 사용하는 데이터를 메모리에 올려놓는 공간)
Undo Log(Undo : 되돌리다) : Rollback과 MVCC에서 사용한다.

중요한 것은 이때 수정된 데이터를 곧바로 원본 DB 파일에 저장하는 것이 아니다.
왜냐하면 DIsk가 너무 느리기에 이걸 업데이트가 일어날 때마다 디스크에 일일이 저장하면 DB가 굉장히 느려진다. 최신 수정된 데이터를 Buffer Pool에 보존해여 계속 캐싱으로 사용하여
DB 성능을 높인다. 최소 수백배~수천배의 차이가 난다.

Q. 자 그런 언제 원본 DB에 반영이 될까?
A. CheckPoint 과정이라고 부르며 나중에 Dirty Page를 Flush하는 과정이 있는데 이때 원본 DB에 반영이 되는 것이다.

Undo Log를 왜 만들까?
1. ROllback을 위해
-> money = 1000를 2000으로 업데이트하면 먼저 Undo Log에 1000이 항상 먼저 저장이 되고 그 다음에 money = 2000이 Buffer Pool에 저장이 된다.
그래야지 ROllback 실행 시 Undo Log에 있는 이전 데이터를 통해서 원복을 한다.

2. MVCC(이게 더 중요)
세션 1이 money=2000을 업데이트하는 트랜잭션을 실행 중며 아직 커밋 안 된 상태라고 하자!(이떄 세션 1은 DB 롹을 획득하고 있다)
근데 세션 2가 이때 money를 SELECT한다고 하자. 해당 데이터에 대해 세션 1이 DB락을 가지고 있기 떄문에  이때 Buffer Pool에 있는 money를 읽는 것이 아니라 Undo Log에 있는 이전
데이터를 보여 줘야 한다. 왜냐하면 아직 세션 1이 최종적으로 commit한 것이 아니기에 도중에 money를 select하게 되면 데이터 동시성 문제가 일어 나기 때문이다. MVCC 기법은 이러한 동시성 문제를
이런 방식으로 해결을 한다.

그럼 UPDATE 실행 시는 어떨까?
먼저 UPDATE 실행 시 해당 원본 데이터를 읽어야 하는데 세션 1이 아직 커밋을 하지 않아 있끼 때문에 원본 데이터를 읽어서 업데이트를 하면 동시성 문제(Multiver Version Concurrency Control은 이 문제를 위해 탄생)가 발생을 한다. 그래서 처음에는 Undo Log에 있는 값을 읽는 다음에 세션 1이
트랜잭션을 종료하여 DB 락을 반환할 때까지 기다린다. DB락이 반환이 되면 세션 2는 DB락을 획득하여 Buffer Pool에 읽는 데이터를 읽어 들여와서 UPdate문을 해결한다.
즉, UPdate는 항상 최신의 데이터를 읽어 들여와서 업데이트를 한다.

당장 Buffer Pool을 사용하냐 아니면 Undo Log를 사용하냐는 해당 데이터에 DB롹이 걸려 있는 지 없는지의 여부로 결정이 된다.
