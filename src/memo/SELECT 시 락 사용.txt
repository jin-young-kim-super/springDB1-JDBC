일반적이 SELECT문은 DB 락을 사용하지 않는다. 
누구나 Buffer Pool이나 원본 데이터 베이스에서 자유롭게 읽어 들일 수가 있다. 

그런데 예외적으로 SELECT ... for UPDATE, 즉 변경하기 위해 읽어 들은 데이터의 경우 commit/rollback을 하기 전에는 원본 데이터를 읽어 들일 수가 없고, DB락을 획득하기 전까지는 다른 세션에서
SELECT를 하여도 DB 락이 반환되어야 지만 Buffer Pool이나 원본 데이터 베이스에 있는 데이터를 읽어 들일 수가 있다.

SELECT ... for UPDATE이 만약 돈을 변경하기 위한 SELECT문일 때 도중에 다른 세션에 의해서 변경이 돼 버리면 절대 안 된다. 
그래서 이떄에는 DB 락이 반환될 떄까지 기다려서 해당 데이터가 최신 commit 상태일 때 읽어 들여야 한다. 
 
※조회 시점에 락이 필요한 경우에는 SELECT FOR UPDATE를 사용하면 된다.
